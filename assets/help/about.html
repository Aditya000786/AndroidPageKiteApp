What is PageKite?

PageKite is a tool which makes your Android a fully fledged part of the World Wide Web. It lets you choose a public name for your device and makes it globally reachable whenever it is powered on and has Internet connectivity.

The app connects one or more server apps to a remote frontend relay server, either one provided by pagekite.net or one you run on your own server.

See below for some examples of server apps which are known to work with PageKite.


Getting Started

Using pagekite.net

  1. Install an SSH or Web Server app
  2. Configure the ports in Local Server Settings and the server app so the ports match.
  3. Sign up for a pagekite.net account and choose a name for your phone, using the built-in signup (or login) option.

Note: if you are using a .pagekite.me name, your websites will automatically have SSL encryption support! Just use the https:// prefix instead of plain HTTP.

Running your own relay

  1. Install an SSH or Web Server app
  2. Configure the ports in Local Server Settings and the server app so the ports match.
  3. Install the Open Source pagekite.py relay software on a computer or VPS with a public IP address and domain name.
  4. Configure the Account Details with your relay machine's public domain name, your shared secret and uncheck the pagekite.net option.

Web and SSH Server Apps

The following apps have been tested and confirmed (as of September 2012) to work with PageKite:

  • SSHDroid by Berserker
    Note that SSH clients must be configured to use HTTP CONNECT. Works well.
  • kWS by Kamran
    A simple, lightweight web server for serving static files. Works well, but sometimes shuts itself off if WiFi is disabled, start up with WiFi off to avoid this problem.
  • PAW Server by Jochen
    A powerful web server for remote-controlling your phone. Works well.
  • IP Webcam by Pavel Khlebovich
    Streaming video and static images work fine, but the app web server generates some unusable direct URLs.

FAQ: Common Problems

Web Server Only Works on WiFi

Unfortunately, some Android web server apps are designed with the assumption that a phone on 3G is completely unreachable. These apps turn themselves off unless WiFi is active. Sadly there is nothing we can do, except contact the authors and ask them to fix their apps.

Most of these apps can be made to work with PageKite (on WiFi only) by selecting Use WiFi IP in the Local Server Settings.

Front Page Works, But Links Fail

Some server apps generate links which include the phone's IP address, which will not work unless your browser is on the same WiFi network. The only solution is to contact the author of the app and politely ask them to fix it so it works with PageKite.

Servers Are Unreachable When Screen is Off

Many Android phones go almost completely to sleep when the screen is off, to conserve battery power. This can be adjusted to some extent in the phone settings, but if you really want your phone to be always available as a web server you will probably need to keep the phone plugged in most of the time.
